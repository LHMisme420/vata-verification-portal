// 04 — How It Works
METHODOLOGY
01
Question Set
30 standardized factual and hallucination detection questions covering history, science, mathematics, and current events. Identical prompts sent to all 6 models via official APIs.
02
Transcript Hashing
Each model's complete response transcript is hashed via SHA-256, producing a unique 256-bit fingerprint that cannot be forged or altered without detection.
03
Merkle Tree
All 6 transcript hashes are assembled into a depth-3 Poseidon hash Merkle tree. The single root value cryptographically commits to all 6 transcripts simultaneously.
04
ZK Proof Generation
A Groth16 zero-knowledge Merkle inclusion proof is generated for each model. This proves membership in the tree without revealing the full transcript contents.
05
On-Chain Verification
All 6 proofs are verified against the deployed Groth16 verifier contract on Ethereum Mainnet. Verification status is permanently recorded on-chain.
06
Immutable Anchoring
The Merkle root is anchored to Ethereum Mainnet with the VATA prefix (0x56415441), creating an immutable timestamp that any party can independently verify forever.
